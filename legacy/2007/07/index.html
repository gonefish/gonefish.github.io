Firefox3对svg的渲染更平滑了

刚才突然看了一个svg格式的图片，在Firefox2下面的渲染速度实在不是行，滚动时非常慢，换Opera9试试后，速度比Firefox2快点。然后我用Firefox3的Nightly测试，发现渲染速度非常好，滚动非常的平滑。接着我又用windows版的Safari查看，渲染速度也非常不错，比Opera9强，但比Firefox3的速度还是差点。在Safari中，好象没有显示滚动条。ie好象需要安装插件才能看。

这非常的Cool，对新技术的支持总是让人兴奋。当然这应该得益于新的gecko内核。

无声的沟通

随着科技的发展，人与人之间的沟通更方便了，但方便的只是沟通工具而已。像短信和IM文字输入方式，这种无声的沟通。总是让我有时候感到不舒服，也许一边在想咒骂你，一边又在和你甜言蜜语。虽然，有时这些仅仅是“赞美”而已。这种无声的沟通让“赞美”变得非常简单，甚至不用你动嘴巴。

其实，我想换一个沟通的方式，因为这样让我感觉到真实。但是，现实并不是这样，这个世界非常古怪，我只能在不舒服的时候睡上一觉，然后让它过去，但决对不会将它忘记。

如果说时间会冲淡一切，所简直是对历史的亵渎。

安全感？

生活中的有些事情，总是那么不让人放心。比如：钥匙串，在我有几次跑进来的时候，钥匙串掉在了地上，如果我没有发现，那么会是非常糟糕的事。这钥匙串让我感到非常不安全，解决方法也许是买一个更好的钥匙串，以至于在你跑动的时候不会掉下来。像我现在在外面租房子住，如果有小偷来光顾，这也给我了一些不安全感。也许这些事情的安全感，都可以通过其它的试为解决，这应该都是钱的问题。

但人与人之间，应该怎么才能建立安全感？如果只是用钱来建立人与人之间的安全感，那这种安全感也太不稳定了。

使用Zend_XmlRpc_Server

前几天在用Django使用MetaWeblog API的实现，发现非常方便。由于我原来用ZF做这个API发现不能使用。一切的原因，可能是我太。。。。

关于Zend_XmlRpc的Debug非常不方便，这让我非常头痛，真的，痛了星期六和星期日。后来使用log后，才发现问题的所在。 在还没有使用log前，我使用Zend_XmlRpc_Client做测试，但使用Zend_XmlRpc_Client后，返回的结果已经自动转换成php类型了。搞得我以为Zend_XmlRpc_Server不会自动转换结果，然后我自己手动转换，这样的话返回的值就错了。直接返回准备好的结果就OK了。

/**
     * The Blogger API getUsersBlogs
     *
     * @param string $appkey
     * @param string $username
     * @param string $password
     * @return array
     */
    public function getUsersBlogs($appkey, $username, $password)
    {
        $authAdapter = new AdminAuth($username, $password);
        $result = $authAdapter->authenticate();

        if ($result->isValid()) {
            $xmls = array(
                array('url' => 'http://'.$_SERVER['SERVER_NAME'], 'blogid' => 1, 'blogName' => Options::title)
            );
            return $xmls;
        }
    }
